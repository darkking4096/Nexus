# Story 6.2: Multi-Profile Dashboard Overview

**Epic**: 6 | **Priority**: MUST | **Owner**: @dev | **Depends On**: 6.1
**Complexity**: 8 pts | **Business Value**: HIGH | **Status**: Draft

## 📋 Descripción

Criar endpoint de visão consolidada que agrega KPIs de todos os perfis conectados. Marina vê com um olhar qual perfil está crescendo, qual está low-engagement, qual precisa atenção. Base para a interface do dashboard.

## 🎯 Critérios de Aceitação

**Given** Marina tem 3+ perfis conectados com dados históricos (>7 dias)  
**When** faz requisição a `GET /api/dashboard/overview`  
**Then**:
- [ ] Response inclui array de perfis com: { id, name, followers, engagement_rate, posts_30d }
- [ ] Total followers e avg engagement calculados corretamente
- [ ] Crescimento semanal calculado (followers_now - followers_7d_ago)
- [ ] Response completa em <500ms

**And** quando ordena perfis por engagement  
**Then**:
- [ ] Top 3 perfis aparecem primeiro
- [ ] Order é persistente entre requests
- [ ] Suporta sorting por: engagement, followers, growth

## 📐 Escopo Técnico

### Endpoint
```
GET /api/dashboard/overview
Response: {
  "profiles": [
    {

      "engagement_rate": 4.2,
      "weekly_growth": 120,
      "posts_30d": 12,

    }
  ],
  "total_followers": 50000,
  "avg_engagement": 4.5,
  "total_posts_30d": 85
}
```

## 🎯 Business Value

Marina vê portfolio inteiro em 1 segundo. Identifica perfis underperforming rapidamente. Permite decisão estratégica: "este perfil precisa mais conteúdo" ou "este está crescendo, replicar estratégia". 30% do valor analytics.

## ⚠️ Risks

1. **Calculation errors em aggregation** (High) — Soma errada = decisões erradas
   - Mitigação: 100% test coverage em cálculos
2. **Performance com 10 perfis** (Medium) — Query lenta
   - Mitigação: Cache 15 min, usar índices DB
