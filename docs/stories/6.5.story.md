# Story 6.5: Performance Report Generation

**Epic**: 6 | **Priority**: SHOULD | **Owner**: @dev | **Depends On**: 6.4
**Complexity**: 10 pts | **Business Value**: MEDIUM | **Status**: Draft

## 📋 Descripción

Gerar relatório consolidado (semanal/mensal) que sumariza performance, compara com período anterior, e identifica principais wins/losses. Marina pode exportar como PDF para mostrar a cliente ("seu Instagram cresceu 15% este mês"). Relatório resume tudo: segue padrões de análise.

## 🎯 Critérios de Aceitação

**Given** um perfil com dados de >30 dias  
**When** Marina requisita `GET /api/reports/{profileId}?period=month`  
**Then**:
- [ ] Response contém período (month ou week), datas de início/fim
- [ ] KPIs principais: followers gained, posts published, avg engagement, reach total
- [ ] Comparação com período anterior: "15% more followers than last month"
- [ ] Top 3 posts do período com métricas
- [ ] Bottom 3 posts (para Marina entender o que não funcionou)

**And** quando exporta como PDF  
**Then**:
- [ ] PDF é gerado em <5s
- [ ] Contém gráficos visuais (growth curve, engagement timeline)
- [ ] Formatado profissionalmente (cor, logo, typography)
- [ ] Adequado para compartilhar com cliente

## 📐 Escopo Técnico


### Dependencies
- Story 6.3 (per-post metrics)
- Story 6.4 (engagement patterns)
- PDF generation library (e.g., pdfkit ou puppeteer)
- Chart library (Chart.js ou similar para visualizações)

## 🎯 Business Value

Marina entrega relatório profissional mensalmente à cliente ("veu seu crescimento aqui"). Prova de valor do NEXUS. Aumenta retenção cliente. 15% do valor analytics.

## ⚠️ Risks

1. **PDF generation performance** (Medium) — Pode ser lenta com muitos gráficos
   - Mitigação: Gerar em background, retornar link após 30s
2. **Gráficos ficam feios se poucos dados** (Low) — Novo perfil pode ter visual ruim
   - Mitigação: Placeholder "não há dados suficientes" se <5 posts
