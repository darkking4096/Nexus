## 📋 Descripción

O curador final do NEXUS: sintetizar insights de engagement (6.4) + relatório (6.5) + dados históricos para gerar recomendações estratégicas via Claude + marketing-instagram-squad. Marina recebe "Publique carousels: seu público engaja 40% mais com esse formato" com impacto estimado. Mais valor = retenção > lucro.

## 🎯 Critérios de Aceitação

**Given** um perfil com histórico completo (>30 posts, >60 dias de dados)  
**When** Marina requisita `GET /api/recommendations/{profileId}`  
**Then**:
- [ ] Response contém array de 5-10 recomendações ordenadas por impacto
- [ ] Cada recomendação tem: priority (high/medium/low), action (string clara), data backing, estimated impact
- [ ] Recomendações são específicas à análise, não genéricas ("post more" é ruim, "post carousels 3x/week at 6pm" é bom)
- [ ] Cada recomendação cita fonte de dado (ex: "baseado em top_hours analysis")
- [ ] Response completa em <10s (pode ser assíncrona)

**And** quando Claude gera recomendações  
**Then**:
- [ ] Prompt inclui contexto: análise do perfil, padrões de engagement, histórico de posts
- [ ] Claude integra frameworks de especialistas (via marketing-instagram-squad: copywriter, content-planner, analyst)
- [ ] Output é validado: recomendações viáveis, específicas, acionáveis

   - Mitigação: Todas recomendações MUST ter data_backing. Prompt específico "cite dados exatos".
2. **Recomendações genéricas/inúteis** (Medium) — "Post more" não ajuda
   - Mitigação: Validação de marketing-squad, rejeitar se não for acionável
3. **Performance** (Medium) — Claude call pode ser lenta (10-30s)
   - Mitigação: Retornar async endpoint, cache por 7 dias, background job

### Implementation Summary
**RecommendationService.ts (Story 6.6 Implementation)**

**Architecture:**
- Claude API integration via @anthropic-ai/sdk (model: claude-opus-4-6)
- 7-day cache for recommendations (7 * 24 * 60 * 60 * 1000 ms)
- Fallback heuristic when Claude fails (no hallucination blocking UX)
- Retry mechanism: if <5 valid recommendations, retry with "Be MORE SPECIFIC" instruction

**Validation Pipeline (5 functions):**
1. `validateRecommendation()` — Schema check (priority, category, action length 20+, data_backing not empty)
2. `isGeneric()` — Regex patterns for generic phrasing (post more, use X, improve, boost, etc.)
3. `isDataGrounded()` — Verify data_backing metrics match profile data (warnings logged)
4. `deduplicateRecommendations()` — Levenshtein similarity >85% = duplicate removal
5. Retry fallback — Max 2 attempts, second with explicit "Be MORE SPECIFIC" instruction

**Prompt Template:**
- Section 1: Role + constraints (specific to THIS profile, generic unacceptable)
- Section 2: Profile data (top content types, peak hours, engagement trends)
- Section 3: Requirements (6 mandatory fields, specificity rules, examples)
- Section 4: Example good vs bad recommendations (calibration)

**Fallback Heuristics (when Claude unavailable):**
- Content type recommendation (if top type >20% higher engagement)
- Posting time recommendation (if peak hour >1.3x average)
- Returns max 2 fallback recommendations (conservative, safe)
