async function randomDelay(min = 1000, max = 5000): Promise<void> {
  const delay = Math.random() * (max - min) + min;
  await new Promise(resolve => setTimeout(resolve, delay));

### 4. Error Handling & Retry Logic

**Cenários**:
- [ ] Login failed (invalid creds, account locked)
- [ ] Rate limited (aguardar 1h ou mais)
- [ ] Network timeout (retry com backoff: 2s, 4s, 8s)
- [ ] Element not found (página mudou, retry)
- [ ] Image upload failed (retry com imagem diferente)

```typescript
async function retryWithBackoff<T>(
  fn: () => Promise<T>,
  maxAttempts = 3,
  initialDelay = 2000
): Promise<T> {
  for (let i = 0; i < maxAttempts; i++) {
    try {
      return await fn();
    } catch (error) {
      if (i === maxAttempts - 1) throw error;
      const delay = initialDelay * Math.pow(2, i);
      await new Promise(resolve => setTimeout(resolve, delay));
    }
  }
  throw new Error('Max retries exceeded');
}
```

## ⚠️ Considerações Especiais

1. **Instagram Anti-Bot**:
   - Instagram detecta automação
   - Delays randômicos + mouse movements ajudam
   - Considerar usar proxies em produção (future enhancement)

2. **Browser Lifecycle**:
   - Abrir browser = 3-5 segundos
   - Considerar manter browser aberto para múltiplas publicações?
   - Implementar timeout (fechar após 30 min inativo)

3. **Credenciais Instagram**:
   - Armazenar com AES-256-GCM
   - NUNCA logar com conta pessoal do dev
   - Usar test account ou conta cliente específica

4. **Rate Limiting**:
   - Instagram limita publicações a ~5-10 por hora por conta
   - Implementar fila de publicação com scheduling
   - Respeitar horários agendados
