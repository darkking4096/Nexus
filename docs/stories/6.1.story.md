# Story 6.1: Analytics Foundation & Instagrapi Integration

**Epic**: 6 | **Priority**: MUST | **Owner**: @dev | **Depends On**: 1.6, 4.5
**Complexity**: 13 pts | **Business Value**: HIGH | **Status**: InProgress

## 📋 Descripción

Implementar integração com Instagrapi para pull de métricas de perfis Instagram. Este é o foundation para todo o módulo de analytics: coleta dados de followers, engagement, métodos de perfil. Base para stories 6.2-6.6.

## 🎯 Critérios de Aceitação

**Given** um perfil de Instagram conectado via Graph API  
**When** o sistema faz requisição de métricas via Instagrapi  
**Then**:
- [ ] Sistema retorna followers count corretamente
- [ ] Sistema retorna engagement rate calculado (comentários + likes / followers)
- [ ] Dados são cacheados por 1h (não refetch do Instagram a cada reload)
- [ ] Erros de rate-limiting são tratados com retry exponencial (3x com backoff)

**And** quando métricas histórico solicitadas  
**Then**:
- [ ] Últimas 30 posts retornam com likes, comments, shares, saves, reach
- [ ] Crescimento de followers por período (semanal) é calculado
- [ ] Timestamps precisos em ISO 8601 UTC

## 📐 Escopo Técnico

### Endpoints
```
GET /api/analytics/{profileId}/metrics
  ├─ current: { followers, engagement_rate, total_reach_30d }
  └─ history: [ { date, followers, engagement_rate } ]

GET /api/content/{contentId}/metrics
  └─ { likes, comments, shares, saves, reach, engagement_rate }

GET /api/analytics/{profileId}/posts
  └─ [ { id, timestamp, media_type, likes, comments, reach } ] (30 últimos)
```

### Dependencies
- Instagrapi Python library (via subprocess or API wrapper)
- SQLite migrations para caching de métricas
- Rate limiting middleware (local queue)

## ⚠️ Risks

1. **Instagrapi rate limiting** (Medium impact) — Instagram limita ~200 calls/hora
   - Mitigação: Implementar cache 1h + queue local + exponential backoff
2. **Dados desincronizados** (Low) — Métricas podem não estar 100% real-time
   - Mitigação: UI mostra "atualizado às 15h23" com refresh manual
3. **Performance com 10+ perfis** (Medium) — Múltiplas requisições simultâneas
   - Mitigação: Usar Promise.all() com limit concorrência 3
