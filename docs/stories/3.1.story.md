### Key Insights
`★ Insight ─────────────────────────────────────`
**Error handling pattern:** When one data source fails (e.g., SearchService), the orchestrator logs a warning but continues with available data. This graceful degradation ensures the research pipeline doesn't break—users get partial results instead of complete failure.

**Caching design:** The 24-hour cache TTL (in ResearchOrchestrator) prevents re-running expensive research repeatedly, while being short enough to catch profile updates. The cache key uses profileId only, making it automatically invalidated on profile ownership changes.
