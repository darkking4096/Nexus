## 📐 Escopo Técnico

### Test Scenarios (Detalhado)
1. **Signup + Login**: Registrar novo usuário, validar email, fazer login, confirmar dashboard carregado
2. **Connect Instagram Profile**: Autorizar app no Instagram (OAuth flow), confirmar profil conectado na UI
3. **Generate Content**: Criar conteúdo (manual), confirmar geração sucesso, validar preview
4. **Approve + Publish**: Aprovar conteúdo gerado, clicar publish, confirmar API response 200
5. **Verify Metrics**: Consultar dados de publicação (likes, comments count), validar sincronização com Instagram API

### Test Account Setup
- Instagram handle: `@synkra-e2e-test` (ou similar)
- Account type: Business ou Creator (requerido para usar Graph API)
- Posts podem ser deletados durante testes
- Rate limiting: 90 requests/15 min (Instagram API) - considerar delays entre testes
- Token refresh: Implementar mock ou real token refresh

### Infrastructure
- Staging environment: `staging.synkra.com` (ou local Docker)
- Database: Reset entre rodadas ou usar transactions
- Instagram Test Account: Manter credenciais em `.env.test`

## ⚠️ Riscos Documentados

| Risco | Impacto | Mitigação |
|-------|---------|-----------|
| Instagram API rate limiting | Testes falham aleatoriamente | Adicionar delays exponenciais + retry logic |
| Test account token expiry | Tests quebram após 60 dias | Implementar token refresh automático ou renovação manual semanal |
| Flakiness de network | False negatives | Usar `waitForNavigation()` + timeout adequado (10s min) |
| Database state inconsistency | Testes dependentes falham em cascata | Isolamento: transaction rollback ou hard reset entre testes |
| Instagram UI changes | Tests quebram sem avisar | Usar locators robustos (data-testid) + monitorar breaking changes |
