**Epic**: 7 | **Priority**: SHOULD | **Owner**: @dev | **Depends On**: 6.1-6.6

## 📋 Descripción

Otimizações sistemáticas em 3 camadas (Backend, Frontend, Database) para atingir métricas de performance críticas: dashboard load < 2s, geração de conteúdo < 30s, queries < 500ms, bundle < 500KB, Lighthouse > 80.

**Validation:**

## 📐 Escopo Técnico

### Backend Optimization
- Query optimization: Implement SELECT specificity, add pagination, batch queries
- Caching strategy: Redis cache para perfis + conteúdo (TTL: 5min para profiles, 1h para content)
- API response compression: Ativar gzip em todas responses
- Connection pooling: Configurar pool size adequado para DB

### Frontend Optimization
- Code splitting: Lazy load components com React.lazy() + Suspense
- Bundle analysis: Usar webpack-bundle-analyzer para identificar duplicatas
- Remove unused dependencies: Audit package.json, remover dev deps de production
- Image optimization: Servir imagens em webp format com fallback
- Minification: Garantir terser está ativo em production build

### Database Optimization
- Index strategy: Criar índices em: `users(id, email)`, `profiles(user_id, instagram_handle)`, `content(user_id, created_at)`, etc.
- Query tuning: Rewrite N+1 queries usando JOINs ou batch loading
- Explain plan review: Validar execution plans para top 10 queries

### Tools & Monitoring
- Lighthouse CI: Integrar em GitHub Actions para validar a cada commit
- Custom metrics: Expor response times via prometheus/grafana para monitoramento contínuo
- Load testing: Simular 100 usuarios simultâneos com k6 ou JMeter
