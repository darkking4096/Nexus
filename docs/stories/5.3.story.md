---

## 📋 Descripción

Extend PublishService to handle Instagram Stories publishing. Stories are ephemeral (24-hour lifespan) vertical format content optimized for mobile viewing. Supports both static images (1080x1920px) and short videos (15s max).

**User Flow:**
1. User selects story content (static image or short video)
2. System validates format: 1080x1920 for images, MP4/WebM for video, max 15s duration
3. Publishes story via PlaywrightService using Instagram story editor
4. Story remains live for 24 hours (Instagram's default)
5. Database updated with story publish URL and timestamp
6. User can track story views and interactions

**Technical Context:**
- Instagram Stories API minimal; Playwright automation required
- Images must be exact 1080x1920 (9:16 aspect ratio)
- Videos max 15s duration
- Story content stored similarly to feed posts but with `type='story'`
- No carousel or text overlay in MVP (future enhancement)

**Validation Method:**

```typescript
private validateStory(imagePath: string): void {
  // 1. Validate dimensions: 1080x1920 (±2% tolerance)
  // 2. Validate format: JPEG or PNG
  // 3. Validate file size: < 4MB
  // 4. Throw error if any validation fails
}

private validateStoryDimensions(width: number, height: number): void {
  // 9:16 aspect ratio = 0.5625
  const aspectRatio = width / height;
  const targetRatio = 9 / 16; // 0.5625
  const tolerance = 0.02; // ±2%
  
  if (Math.abs(aspectRatio - targetRatio) > tolerance) {
    throw new Error(`Invalid story aspect ratio: ${aspectRatio.toFixed(2)}, expected 9:16`);
  }
  
  // Check dimensions within reasonable bounds of 1080x1920
  if (Math.abs(width - 1080) > 20 || Math.abs(height - 1920) > 20) {
    console.warn(`Story image not exactly 1080x1920 (got ${width}x${height}), but aspect ratio valid`);
  }
}
```

```typescript
// Validate story image (dimensions, format, size)
private validateStory(imagePath: string): void {
  this.validateStoryFile(imagePath);
  this.validateStoryDimensions(imagePath);

private validateStoryFile(imagePath: string): void {
  const stats = fs.statSync(imagePath);
  const ext = path.extname(imagePath).toLowerCase();
  
  if (stats.size > 4 * 1024 * 1024) throw new Error('Story image > 4MB');
  if (!['.jpg', '.jpeg', '.png'].includes(ext)) throw new Error('Invalid format (JPEG/PNG only)');
}

private validateStoryDimensions(imagePath: string): void {
  const { width, height } = getImageDimensions(imagePath);
  const aspectRatio = width / height;
  const targetRatio = 9 / 16; // 0.5625
  const tolerance = 0.02;
  
  if (Math.abs(aspectRatio - targetRatio) > tolerance) {
    throw new Error(`Invalid story aspect ratio (${aspectRatio.toFixed(3)}), expected 9:16`);
  }
}
```

## ⚠️ Technical Risks & Notes

**Risk 1: Story Dimension Variance**
- Different devices/cameras produce slightly different dimensions
- Instagram might auto-resize stories
- Mitigation: Use ±2% tolerance on aspect ratio, not exact pixel matching
- Resolution: Test with various image sources; log warnings for near-valid dimensions

**Risk 2: Instagram Story UI Differs by Device**
- Instagram regularly updates UI; "Your Story" button location might change
- Mitigation: PlaywrightService should be resilient to UI changes (use role selectors, not pixel positions)
- Resolution: Monitor Playwright logs; update selectors if UI changes

**Risk 3: Story Analytics**
- Instagram doesn't expose story view counts via API
- Stories are ephemeral; data lost after 24h
- Mitigation: For MVP, don't track story metrics; future story (Story 5.6?) can add analytics
- Resolution: Document limitation; provide user with manual Instagram Insights

**Risk 4: Auto-Deletion**
- Stories auto-delete after 24h; no database cleanup needed
- But user might see "published story" indefinitely in history
- Mitigation: Add optional manual delete endpoint (future story)
- Resolution: Document 24h lifespan in UI
