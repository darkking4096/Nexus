## 📋 Problema & Necessidade

Story 3.5 implementou caption generation com branding (hooks, CTAs, narrativas), mas **falta a component de hashtag generation**. Hashtags são críticas para descobribilidade no Instagram:
- Nicho-específicos (audience target)
- Trending (viral potential)
- Balanced mix (not just top 30, not just long-tail)

Usuários precisam de **hashtag suggestions otimizadas** que:
- Respeitam restrição Instagram (30 max por post)
- Evitam hashtags shadowbanned
- Mostram volume/competition (meta)

**Valor de negócio:** Hashtags corretas podem aumentar reach em 2-5x. Geração automática remove manual curation.

## 🎯 Critérios de Aceitação

### AC-1: Geração de Hashtags (10-15)
```gherkin
Given um perfil Instagram com análise completada (Story 3.4)
When POST /api/content/generate-hashtags
  { "profile_id": "inst_123", "content_type": "carousel|story|reel", "analysis": {...} }
Then recebo HTTP 200
And response contém array de 10-15 hashtags
And cada hashtag é string sem espaços, começa com #
And inclui metadata: { hashtag, volume, competition, recommendation_score }
```

### AC-2: Relevância ao Nicho
```gherkin
Given um perfil com nicho bem definido (e.g., "fitness", "moda", "culinária")
When gero hashtags
Then:
  - Todos os hashtags são relevantes ao nicho
  - No máximo 2 hashtags completamente genéricos (ex: #love, #instagood)
  - Mínimo 8 hashtags nicho-específicos (ex: #fitnessmotivation para fitness)
  - recommendation_score > 0.6 para todos
```

### AC-3: Mix Trending + Niche-Specific
```gherkin
Given análise de trending (via Story 2.1 - Pesquisa)
When gero hashtags
Then:
  - 5-6 trending hashtags (volume > 100K, recency < 7 dias)
  - 5-6 niche hashtags (volume 10K-100K, high relevance)
  - 2-3 long-tail hashtags (volume < 10K, ultra-specific)
  - Distribution equilibrada: não só top 30, não só obscuro
```

### AC-4: Validação de Shadowban
```gherkin
Given lista gerada de hashtags
When aplico validação
Then:
  - Crosscheck contra lista de shadowbanned hashtags (atualizada semanal)
  - Flag quaisquer hashtags suspeitos com warning
  - Remover automaticamente se relevance_score < 0.3
  - Logar hashtags removidos para analytics
```

### AC-5: Testes Implementados
```gherkin
Given implementação completa
Then testes cobrem:
  - [x] Geração com diferentes nichos (5+ nichos testados)
  - [x] Mix trending/niche distribution
  - [x] Shadowban detection (mock list)
  - [x] Error handling (missing analysis, invalid profile)
  - [x] Metadata accuracy (volume, competition, score)
  - [x] Integração com análise de tendências (Story 2.1)
```

## 📚 Dev Notes

### Procedimento de Implementação
1. Carregar análise do perfil de Story 3.4 (nicho, audience, vibe)
2. Chamar Story 2.1 (trend research) para trending hashtags da semana
3. Combinar com hashtag database (nicho-specific, indexado por category)
4. Calcular recommendation_score baseado em:
   - Relevância ao nicho (0-1)
   - Volume adequado para account size (0-1)
   - Recency se trending (0-1)
   - Não-shadowbanned (bin: 0 or 1)
5. Selecionar top 10-15 por score
6. Validar contra shadowban list (semanal update)
7. Retornar com metadata

### Pontos Críticos
- **Data Currency:** Trending hashtags mudam diariamente. Cache max 1 dia.
- **Shadowban List:** Manter lista atualizada (semanal). Usar fonte confiável (ex: Instagram engagement communities).
- **Account Size:** Recomendação muda para micro (< 10K) vs macro (> 100K). Considerar followers.
- **Rate Limit:** Story 2.1 (trend research) pode ter rate limiting. Implementar caching.
- **Cold Start:** Novo perfil sem análise → usar defaults genéricos do nicho.

### Post-QA Bugs Fixed (2026-04-13)

Após análise detalhada do código pós-QA gate, foram identificadas e corrigidas 5 inconsistências reais:

1. **Issue 4 (Alta):** Niche `beauty` ausente de `localNicheDatabase` em `TrendingHashtagClient.ts`
   - Causa: AC-5 testa 6 niches mas `beauty` retornava array vazio
   - Fix: Adicionado bloco `beauty` com 10 hashtags no `localNicheDatabase`

2. **Issue 2 (Média):** Mutação `status: 'removed'` em objetos fora do array retornado
   - Causa: `finalHashtags = hashtags.slice(0, maxHashtags)` antes da mutação loop
   - Problema: `totalRemoved` fica não-zero mas nenhum hashtag retornado tem `status: 'removed'`
   - Fix: Removido bloco orphaned `if (removedCount > 0) { ... }`

3. **Issue 1 (Média):** `minHashtags` ignorado completamente
   - Causa: Rota valida `min_hashtags >= 10` mas `generateHashtags()` desestrutura apenas `maxHashtags`
   - Fix: Adicionado `minHashtags = 10` à desestruturação + validação com logger.warn

4. **Issue 3 (Baixa):** `totalFlagged` sempre retorna 0
   - Causa: `totalFlagged: 0` hardcoded; `shadowbanAlerts` já contém o valor correto
   - Fix: Trocar `totalFlagged: 0` por `totalFlagged: shadowbanAlerts`

5. **Issue 5 (Cosmética):** `checkShadowbanListHealth()` acessado via bracket notation
   - Causa: `generator['checkShadowbanListHealth']()` em rota e testes
   - Fix: Adicionar `public` explícito + trocar bracket notation por dot notation

**Testes adicionados:**
