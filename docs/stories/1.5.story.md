### Key Implementation Decisions

**Graceful EXA Degradation**: Service was designed to work WITHOUT EXA MCP configured. When @devops enables EXA, SearchService will automatically call it. Currently uses mock search results for development/testing.

**Pattern**: Service checks `hasEXA` flag at runtime, tries real search first, falls back to mock if not available. This allows:
- Development without MCP infrastructure
- Testing with predictable mock data
- Easy flip to real search when EXA configured
