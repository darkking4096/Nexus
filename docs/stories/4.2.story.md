# Story 4.2: Approval UI & UX

**Epic**: 4 | **Priority**: MUST | **Owner**: @ux-design-expert | **Depends On**: 4.1 | **Status**: InProgress | **Validated**: ✅ 2026-04-13

## 📋 Descripción

Design e implementação de UI para aprovações em manual mode. Interface responsiva, acessível e intuitiva para revisão e edição de conteúdo em cada etapa do workflow. Componentes reutilizáveis para ApprovalCard, EditForm e PreviewPanel.

## 🎯 Critérios de Aceitação

### Acceptance Criteria (Given/When/Then Format)

**AC 1: Design system aprovado**
```
Given: Figma design file com componentes de aprovação
When: @ux-design-expert submete para design review
Then: Design é aprovado por @product-team com feedback zero-changes necessário
```

**AC 2: ApprovalCard component renderiza corretamente**
```
Given: ApprovalCard recebe props (step, title, content, status)
When: Component renderiza
Then: Exibe título, conteúdo, status badge, e action buttons (approve/reject/edit)
```

**AC 3: EditForm permite edição de caption**
```
Given: Usuario em step "caption"
When: Clica no botão edit ou edita texto inline
Then: Form mostra texto editável, botões save/cancel, e Preview updates em real-time
```

**AC 4: PreviewPanel mostra preview do conteúdo**
```
Given: Conteúdo em qualquer step
When: PreviewPanel é renderizado
Then: Exibe visual e caption em preview mode (read-only), simulando como ficaria publicado
```

**AC 5: Design é 100% responsivo**
```
Given: Suite de testes responsivos (mobile 375px, tablet 768px, desktop 1024px+)
When: Cada breakpoint é testado
Then: Layout ajusta sem scroll horizontal, todos os buttons/forms são acessíveis
```

**AC 6: Acessibilidade WCAG 2.1 Level AA**
```
Given: Components renderizados
When: Executado Axe accessibility scan
Then: Zero violations em Level AA (ARIA labels, contrast ratios 4.5:1+, keyboard navigation funcional)
```

**AC 7: Testes unitários para todos os componentes**
```
Given: Suite de unit tests para ApprovalCard, EditForm, PreviewPanel, ApprovalToolbar
When: npm test é executado
Then: Todos os testes passam, coverage >= 85% por componente
```

## 📐 Escopo

### IN (O que está incluído)
- 4 componentes React: ApprovalCard, EditCaptionForm, PreviewPanel, ApprovalToolbar
- Design system aprovado (Figma)
- Responsive CSS/Tailwind para 3 breakpoints (mobile, tablet, desktop)
- WCAG 2.1 Level AA compliance (accessibility)
- Unit tests com 85%+ coverage
- Integração visual com approval panel backend (Story 4.1)
- Storybook stories para documentação de componentes

### OUT (O que NÃO está incluído)
- API integration (delegado para Story 4.1)
- Mode toggle UI (delegado para Story 4.6)
- Theming/dark mode (future feature)
- Advanced animations (out of scope)

## 💼 Valor de Negócio

Interface intuitiva e acessível reduz friction para usuários aprovarem conteúdo, acelera publication cycle, e garante compliance com accessibility standards. Componentes reutilizáveis facilitam manutenção futura e expansion de features de approval.
