# Story 8.1.3: Frontend Vercel Deploy & CI/CD

**Epic**: 8.1 | **Priority**: MUST | **Owner**: @devops | **Depends On**: None (can run in parallel with 8.1.2)  
**Status**: Ready | **Complexity**: M (8 points - MEDIUM risk, multi-phase setup)

## 📋 Description

Connect GitHub repository to Vercel, configure build settings and deployment automation (Phase 1). Phase 2 performance optimization/monitoring is in Story 8.1.3b. This story covers: Vercel project creation, build command configuration, environment variable setup, CI/CD automation for main branch deployments, and preview deployments for pull requests.

**As a** operations engineer,  
**I want** to deploy the frontend to Vercel with automatic CI/CD,  
**so that** users can access the application via a public URL with zero-downtime deployments and automatic rollback on failure.

## 🎯 Acceptance Criteria

**Vercel Project Setup:**
- [ ] GitHub repository connected to Vercel (OAuth or personal access token)
- [ ] Vercel project created with project name and team configuration
- [ ] Root directory specified correctly (monorepo: `packages/frontend`)
- [ ] Build command configured: `npm run build` (must succeed locally first)
- [ ] Output directory configured: `dist/` (from Vite build)

**Build Configuration & Validation:**
- [ ] Build command verified to work locally (npm run build runs without errors)
- [ ] npm run typecheck passes in production build
- [ ] npm run lint passes in production build
- [ ] Build artifacts verified in Vercel dashboard
- [ ] Vercel preview deployment accessible and functional

**Environment Variables & API Integration:**
- [ ] `.env.production` configured in Vercel project settings (not committed to git)
- [ ] API endpoint URL (VITE_API_URL) points to backend
- [ ] Environment variable validation on startup
- [ ] Secret variables never logged (no credentials exposed)
- [ ] API calls from preview deployment to backend successful

**CI/CD Automation:**
- [ ] Auto-deployment enabled on main branch push
- [ ] Preview deployments enabled for pull requests
- [ ] Build logs accessible in Vercel dashboard
- [ ] Deployment triggered automatically within 5 minutes of push
- [ ] Deployment notifications sent to team (Slack/email)

**Rollback & Deployment Safety:**
- [ ] Rollback procedure documented
- [ ] One-click rollback verified in Vercel dashboard
- [ ] Previous deployment versions retained (> 5 recent versions)
- [ ] Rollback time verified (< 2 minutes)

## 🧠 Dev Notes

### Current Frontend Context

The frontend is built with React + Vite and located at:
- **Root**: `packages/frontend/`
- **Build Command**: `npm run build` (must work locally first)
- **Output**: `dist/` directory
- **API Calls**: All requests to backend at localhost:5000 (to be updated)

### Deployment Strategy

1. **Phase 1: Local Build Validation** — Ensure `npm run build` succeeds locally
2. **Phase 2: Vercel Project Setup** — Connect GitHub and create Vercel project
3. **Phase 3: Build Configuration** — Configure build command and output directory
4. **Phase 4: Environment Setup** — Add environment variables for all environments
5. **Phase 5: Deployment** — Trigger first deployment and validate
6. **Phase 6: Performance Monitoring** — Measure Core Web Vitals and establish baseline

### Key Technical Decisions

- **Deployment Platform**: Vercel (optimized for Next.js/React, excellent performance)
- **Build System**: Vite (fast, already in use)
- **API Endpoint**: Update to use environment variable (VITE_API_URL)
- **Monitoring**: Vercel Analytics + performance monitoring
- **Regions**: US-East primary, EU fallback
- **Preview Deployments**: Enable for all PRs

### Vite Build Configuration

Frontend uses Vite with:
- **Build Output**: `dist/`
- **Assets**: Served from `/` (root path)
- **Environment Variables**: VITE_* prefix required
- **API URL**: Must be set via environment variable, not hardcoded

### Frontend to Backend Communication

API endpoint configuration must support:
- Local development: `http://localhost:5000`
- Staging: Backend staging URL (TBD by Story 8.1.2)
- Production: Backend production URL

CORS must be configured to allow frontend domain.

### Testing Standards

- **Build Verification**: `npm run build` runs without errors or warnings
- **Type Checking**: `npm run typecheck` passes
- **Linting**: `npm run lint` passes
- **Preview Deployment**: Vercel preview URL is functional
- **API Integration**: API calls to backend successful from preview/production
- **Performance**: Core Web Vitals meet targets

### Important References

- Vercel Documentation: https://vercel.com/docs

- Environment Variables: https://vercel.com/docs/concepts/projects/environment-variables
