# Story 5.2: Carousel Publishing (10+ Image Support)

**Epic**: 5 - Publishing Pipeline | **Status**: Draft | **Priority**: MUST | **Owner**: @dev | **Depends On**: 1.3, 5.1

**Date Created**: 2026-04-14  
**Complexity**: Medium (Image validation logic; carousel routing already in 5.1)  
**Business Value**: Enable users to publish high-impact carousel content (10+ slides) with proper formatting, transition effects, and Instagram optimization.

---

## 📋 Descripción

Extend PublishService to handle carousel publishing with support for 10+ slide carousels. Instagram carousels have special requirements: proper slide ordering, aspect ratio consistency (1.91:1 or 4:5), and optional transition effects.

**User Flow:**
1. User selects carousel content with multiple images
2. System validates carousel structure (10+ images, correct aspect ratios)
3. Publishes carousel via PlaywrightService using automated slide navigation
4. Applies anti-bot delays between slide uploads
5. Uses retry logic with exponential backoff on failures
6. Database updated with all slide IDs and carousel post URL

**Technical Context:**
- Story 5.1 handles single-photo publish; this story extends carousel support
- Carousel images stored as JSON array in `content.carousel_json`
- Each slide needs validation: dimensions, file size, format
- PlaywrightService must handle multi-slide automation

**Helper Methods:**
```typescript
// Extract carousel image paths from carousel_json
private getCarouselPaths(content: ContentData): string[] {
  const carousel = JSON.parse(content.carousel_json);
  return carousel.images || [];
}

// Validate carousel structure and images
private validateCarousel(imagePaths: string[], carousel_json: string): void {
  if (imagePaths.length < 2) throw new Error('Carousel needs at least 2 images');
  if (imagePaths.length > 10) throw new Error('Instagram allows max 10 carousel slides');
  
  for (const path of imagePaths) {
    this.validateImageFile(path);
    this.validateImageDimensions(path);
  }
}

// Check image dimensions for carousel aspect ratio
private validateImageDimensions(imagePath: string): void {
  const { width, height } = getImageDimensions(imagePath);
  const aspectRatio = width / height;
  
  // Allow 1.91:1 (1.910) or 4:5 (0.800)
  const ratio191 = Math.abs(aspectRatio - 1.91) < 0.095; // ±5%
  const ratio45 = Math.abs(aspectRatio - 0.8) < 0.04;    // ±5%
  
  if (!ratio191 && !ratio45) {
    throw new Error(`Invalid aspect ratio ${aspectRatio}: must be 1.91:1 or 4:5`);
  }
}

// Validate image file (size, format)
private validateImageFile(imagePath: string): void {
  const stats = fs.statSync(imagePath);
  const ext = path.extname(imagePath).toLowerCase();
  
  if (stats.size > 8 * 1024 * 1024) throw new Error('Image > 8MB');
  if (!['.jpg', '.jpeg', '.png'].includes(ext)) throw new Error('Invalid format (JPEG/PNG only)');
}
```

## ⚠️ Technical Risks & Notes

**Risk 1: Instagram Carousel Format Requirements**
- Instagram has strict requirements for carousel slides
- Mitigation: Validate dimensions before upload (fail fast)
- Resolution: Documentation in code comments; test with real Instagram account

**Risk 2: Multi-Slide Upload Timing**
- Uploading 10 images sequentially might take 2-3 minutes
- Browser might timeout if delays are too long
- Mitigation: Use reasonable delays (1-5s) and test with 10-slide carousel
- Resolution: Adjust delay constants if needed; monitor logs

**Risk 3: Carousel Post ID Extraction**
- Instagram might not return individual slide IDs
- Mitigation: Primary goal is carousel post ID; slide IDs are optional
- Resolution: Store what's available; document in response

**Risk 4: Aspect Ratio Validation**
- Image library might give slightly different dimensions than actual
- Mitigation: Use ±5% tolerance on aspect ratio checks
- Resolution: Test with various image sources; adjust tolerance if needed
