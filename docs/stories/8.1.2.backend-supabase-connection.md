## 🧠 Dev Notes

### Current Backend Context

The Express API is located in `packages/backend/` and currently uses:
- **Driver**: better-sqlite3 (synchronous, file-based)
- **Connection**: Direct file descriptor to SQLite database
- **Pattern**: Service filtering (`.eq('service', 'ttcx')`) used throughout

All existing Express routes (30+ services) expect consistent database interface.

### Migration Strategy

1. **Phase 1: Dependency Swap** — Install pg, remove better-sqlite3
2. **Phase 2: Connection Module** — Create new database connection module with connection pooling
3. **Phase 3: Query Wrapper** — Implement query execution interface compatible with existing code
4. **Phase 4: Testing** — Verify all existing API endpoints work with Supabase
5. **Phase 5: Environment Setup** — Configure .env files for all environments

### Key Technical Decisions

- **PostgreSQL Client**: Use `pg` library (stable, widely used, good connection pooling)
- **Connection Pool**: Create single pool instance at application startup
- **Query Interface**: Wrap pg client to maintain compatibility with existing query patterns
- **Error Handling**: Implement circuit breaker pattern for database unavailability
- **Configuration**: Use dotenv for local development, environment variables in production

### Existing Query Patterns to Preserve

The backend uses queries like:
```javascript
db.prepare("SELECT * FROM users WHERE service = ?").all('ttcx')
db.exec("INSERT INTO profiles ...")
db.transaction(() => { ... })
```

These must continue working with new PostgreSQL connection.

### Testing Standards

- **Test Framework**: Vitest (same as existing tests)
- **Test Location**: `packages/backend/tests/db/`
- **Pattern**: Integration tests that verify connection, query execution, and error handling
- **Fixtures**: Use test Supabase database for integration testing
- **Mocking**: Mock environment variables, test timeout scenarios

### Important References

- PostgreSQL pg library: https://node-postgres.com/
- Connection pooling best practices: https://node-postgres.com/features/pooling
- Environment management: https://github.com/motdotla/dotenv
- Error handling patterns: https://node-postgres.com/docs/apis
