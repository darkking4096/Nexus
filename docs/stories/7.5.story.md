# Story 7.5: User Onboarding Flow & Interactive Tutorial

**Epic**: 7 | **Priority**: SHOULD | **Owner**: @ux-design-expert | **Depends On**: 2.1 (Dashboard UI complete)  
**Status**: Draft | **Complexity**: M (Medium)

## 📋 Descripción

Flow de onboarding guiado: welcome screen → step-by-step tutorial (5-10 min) → profile setup wizard → context-aware help tooltips. Otimizado para reduzir abandono e acelerar time-to-first-publish.

## 🎯 Critérios de Aceitação

**Welcome Screen:**
- [ ] Exibido apenas na primeira visita (flag: `hasCompletedOnboarding`)
- [ ] Título + descrição clara de benefícios principais
- [ ] CTA (Call-to-Action): "Start Tutorial" button bem visível
- [ ] Skip option disponível (para power users)
- [ ] Visual atrativa (hero image ou animated SVG)
- [ ] Mobile-responsive (testar em iPhone + Android)

**Tutorial (5-10 min interativo):**
- [ ] Step 1: "Create Your Profile" (name, bio, profile picture)
- [ ] Step 2: "Connect Instagram" (OAuth flow com visual guide)
- [ ] Step 3: "Generate Your First Post" (demo com template pré-preenchido)
- [ ] Step 4: "Publish & Share" (botão publish highlighted)
- [ ] Step 5: "View Analytics" (preview dashboard com dados demo)
- [ ] Progress bar: mostra posição (ex: 2/5)
- [ ] Back button: navegar entre steps
- [ ] Auto-save: dados não são perdidos se user voltar
- [ ] Timing: < 10 minutos pra completar (medido)

**Profile Setup Wizard:**
- [ ] Form fields: Name, Bio, Profile Picture (obrigatórios)
- [ ] Instagram connection: OAuth flow integrado
- [ ] Validation inline: feedback imediato de erros
- [ ] Success confirmation: "Profile created!"
- [ ] Skip option: "Set up later" (salva partial data)

**Help Tooltips:**
- [ ] Contextuais: aparecem em hover ou click em ?
- [ ] Não intrusive: não bloqueiam interação
- [ ] Exemplos: "Instagram Handle: @yourname (without @)"
- [ ] Cobertura: tooltips em todos 5 steps + dashboard principal
- [ ] Dismissable: user pode fechar com X
- [ ] Mobile: taps instead of hover (touch-friendly)

**Testing & Validation:**
- [ ] Unit tests: onboarding state (completion, step navigation)
- [ ] Integration tests: OAuth flow, profile save
- [ ] E2E test: full onboarding flow (manual)
- [ ] A/B test ready: infrastructure para testar variants
- [ ] Accessibility: WCAG 2.1 AA compliant (screen reader support)
- [ ] Performance: First paint < 1s, transitions smooth (60fps)

## 📐 Escopo Técnico

### Frontend Components
- `OnboardingFlow.tsx` — Main orchestrator, state management
- `WelcomeScreen.tsx` — First-time visitor landing
- `TutorialStep.tsx` — Reusable step component
- `ProfileSetupForm.tsx` — Form fields + validation
- `InstagramConnectFlow.tsx` — OAuth integration
- `TooltipProvider.tsx` — Context provider para tooltips
- `ProgressBar.tsx` — Step progress indicator

### State Management
- Redux / Context: track `onboardingStep`, `profileData`, `isCompleted`
- Local storage: cache incomplete profile data (recovery if closed)
- Backend: save profile after wizard completion

### Data Model
```javascript
// Onboarding state
{
  userId: "uuid",
  step: 1-5,
  profileData: {
    name: "string",
    bio: "string",
    profilePicture: "url"
  },
  instagramConnected: boolean,
  isCompleted: boolean,
  completedAt: "timestamp"
}
```

### Styling & Design
- Design system: Reuse existing button, input, modal components
- Colors: Brand colors (primary CTA, secondary actions)
- Icons: Feather icons or custom SVGs
- Animations: Smooth transitions between steps (300-500ms)
- Dark mode: Responsive to system preference

## ⚠️ Riscos Documentados

| Risco | Impacto | Mitigação |
|-------|---------|-----------|
| Tutorial muito longo | High abandonment | Manter < 10 min, measure completion rate |
| OAuth flow brittle | Users cannot connect | Extensive testing em staging, error handling robusto |
| Mobile layout issues | Bad experience on phones | Test early on real devices (iPhone, Android) |
| Data loss during signup | Users frustrated | Auto-save em local storage, recovery mechanism |
| Accessibility issues | Users excluded | WCAG 2.1 AA audit before launch |
| Skip option leads to confusion | High support load | Show "Complete onboarding" CTA in dashboard |
