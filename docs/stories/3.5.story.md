# Story 3.5: Content Generation Engine (Captions & Hooks)

**Epic**: 3 — NEXUS Intelligence Pipeline | **Priority**: MUST | **Owner**: @dev | **Depends On**: 3.4 ✅

**Status**: ✅ Done | **Complexity**: M (Medium) | **Points**: 8 | **Sprint**: TBD

---

## 📋 Problema & Necessidade

A plataforma NEXUS coleta análise comportamental do perfil Instagram (Story 3.4) mas não consegue **converter insights em conteúdo criativo**. Usuários precisam de múltiplas opções de caption (hooks, CTAs, teaser, narrativas) que respeitem o **tom da marca** e o **calendário de publicação** da conta.

**Valor de negócio:** Reduzir tempo de criação de conteúdo de 30min → 5min (automação de 85% da escrita).

---

## 🎯 Critérios de Aceitação

### AC-1: Endpoint Funcional
```gherkin
Given um perfil Instagram com análise completada (Story 3.4)
When POST /api/content/generate-caption
  { "profile_id": "inst_123", "analysis": {...}, "caption_types": ["hook", "cta"] }
Then recebo HTTP 200
And response contém { "captions": [...], "metadata": {...} }
```

### AC-2: Geração com Múltiplas Opções
```gherkin
Given um pedido de geração de caption
When processado pelo @copywriter da marketing-squad
Then retorna EXATAMENTE 3 opções de caption (hook, teaser, CTA)
And cada opção tem 50-150 caracteres
And cada opção inclui hashtags relevantes (3-5)
```

### AC-3: Frameworks Customizáveis
```gherkin
Given um perfil com brand tone definido (e.g., "casual", "profissional", "viral")
When gero captions
Then aplico framework específico:
  - "casual" → Tom amigável, emojis, calls-to-action leves
  - "profissional" → Linguagem formal, CTAs diretas, sem emojis
  - "viral" → Provocador, pergunta retórica, FOMO
```

### AC-4: Integração com Marketing Squad
```gherkin
Given acesso ao @copywriter squad
When gero caption
Then copywriter-agent refina resultado para qualidade editorial
And retorna com score de confiança (0-100)
```

### AC-5: Testes Implementados
```gherkin
Given implementação completa
Then testes cobrem:
  - [x] Geração com perfis sem análise (erro gracioso)
  - [x] Diferentes brand tones
  - [x] Limites de caracteres respeitados
  - [x] Hashtags válidas (conforme políticas Instagram)
  - [x] Integração com marketing-squad
```

## 📚 Dev Notes

### Procedimento de Implementação
1. Ler análise gerada em 3.4 (schema em `docs/architecture/NEXUS-schema.md`)
2. Estruturar prompts para copywriter agent (templates em `squads/marketing-instagram-squad/`)
3. Implementar retry logic com exponential backoff (API pode ter rate limiting)
4. Validar saída contra regras Instagram (caracteres, hashtags, palavras-chave blocked)

### Pontos Críticos
- **Rate limiting:** Copywriter agent pode estar overloaded — implementar fila (queue)
- **Qualidade:** Se confidence score < 50%, retornar erro e logar para manual review
- **Latência:** Alvo: < 3s por geração (incluindo squad call)

### Referências
- Marketing Squad: `squads/marketing-instagram-squad/README.md`
- Análise schema: `docs/architecture/NEXUS-schema.md`

## 🚨 Riscos & Mitigações

| Risco | Severity | Mitigação |
|-------|----------|-----------|
| Copywriter squad offline | HIGH | Implementar fallback para templates estáticos + manual review queue |
| Captions com qualidade baixa | HIGH | Implementar confidence scoring, rejeitar score < 50% |
| Rate limit da API Claude | MEDIUM | Queue com backoff exponencial, retry até 3x |
| Hashtags flagged como spam | MEDIUM | Validar contra lista de blocked hashtags (atualizar semanal) |
| Latência > 5s | MEDIUM | Implementar cache por brand tone, log de slowness |
