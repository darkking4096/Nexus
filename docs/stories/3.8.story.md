## 📋 Problema & Necessidade

A plataforma NEXUS pode gerar imagens para feed posts (Story 3.6), mas **não consegue criar Stories** (formato 1080x1920 vertical) com **identidade visual do perfil**. Stories têm dimensões diferentes e exigem brand consistency (cores, fonts, logos). Usuários precisam de automação para:
- Geração rápida de story frames (1-5s)
- Aplicação automática de brand colors (não apenas content, mas visual style)
- Text overlay otimizado para vertical format (readable, positioned)

**Valor de negócio:** Stories are shorter-lived but higher engagement. Gerar 1 story por dia = 7 stories/semana automáticas (vs manual creation 20-30min per story).

## 🎯 Critérios de Aceitação

### AC-1: Geração de Story Frame (1080x1920)
```gherkin
Given um request com conteúdo de story
When POST /api/visual/generate-story
  { "profile_id": "inst_123", "content": {...}, "brand_id": "brand_456" }
Then recebo HTTP 200
And response contém buffer PNG (1080x1920px, RGB, 8-bit)
And imagem pronta para upload direto no Instagram
```

### AC-2: Brand Colors Application
```gherkin
Given brand palette (primary, secondary, accent, background)
When renderizo story
Then:
  - Background usa brand colors (gradient ou solid baseado em brand config)
  - Text color has contraste 4.5:1 contra background (WCAG AA)
  - CTAs use brand accent color
  - Subtle brand logo/watermark no canto (if brand config has logo)
```

### AC-3: Text Overlay Otimizado
```gherkin
Given headline (max 40 chars) + body (max 150 chars) + CTA
When aplico text overlay
Then:
  - Headline: bold, centered, 32px font
  - Body: regular, left-aligned, 18px font
  - CTA: button-style, brand color, centered, 16px font
  - Tudo readable em mobile (min 16px effective size)
  - Line height 1.5 (accessibility)
```

### AC-4: Brand Consistency
```gherkin
Given múltiplas stories geradas para um perfil
When renderizo 5 stories em sequência
Then:
  - Cada story mantém identidade visual coerente
  - Colors consistentes
  - Font sizes consistentes
  - Layout positioning consistente
```

### AC-5: Testes Implementados
```gherkin
Given implementação completa
Then testes cobrem:
  - [x] Geração com conteúdo mínimo (headline only)
  - [x] Geração com conteúdo máximo (headline + body + CTA)
  - [x] Brand color application (5 different brand palettes)
  - [x] Text contraste validation (passing + failing)
  - [x] Error handling (missing brand, invalid dimensions)
  - [x] File size < 500KB (Instagram optimal)
```

## 📚 Dev Notes

### Procedimento de Implementação
1. Carregar brand config (colors, logo, fonts) baseado em `brand_id`
2. Usar Canvas/Sharp para criar background (solid ou gradient)
3. Aplicar SVG overlay com text (headline, body, CTA)
4. Validar contraste WCAG AA pix-by-pix
5. Embed brand logo no canto (se disponível)
6. Retornar buffer PNG otimizado para Instagram

### Pontos Críticos
- **Dimensions:** Instagram Story é exatamente 1080x1920. Sem resize automático — falhar se input errado.
- **Contraste:** WCAG AA 4.5:1 obrigatório. Se falhar, usar white/black background semi-transparente.
- **Font Rendering:** Node-canvas precisa de system fonts. Usar fallback para "sans-serif".
- **File Size:** Instagram aceita max 8MB, mas otimizar para < 500KB (PNG).
- **Brand Logo:** Se dimensões inadequadas, escalar ou skip com warning.

### Referências
- Brand config schema: `docs/architecture/NEXUS-schema.md` (brands table)
- Instagram story specs: `docs/guides/instagram-best-practices.md`
- WCAG contrast: https://www.w3.org/WAI/WCAG21/Understanding/contrast-minimum.html
- Sharp PNG optimization: https://sharp.pixelplumbing.com/api-output#png
- Node-canvas fonts: https://github.com/Automattic/node-canvas#text

### Dependências
- `sharp` (background generation, final PNG output)
- `node-canvas` (text rendering, SVG overlay)
- `color-contrast-checker` (validation)

## 🚨 Riscos & Mitigações

| Risco | Severity | Mitigação |
|-------|----------|-----------|
| Brand logo grande demais | MEDIUM | Validar dimensions, escalar automaticamente ou skip com warning |
| Contraste falha em backgrounds escuros | HIGH | Aplicar white/semi-transparent background + re-validate |
| Text cuts off em mobile preview | MEDIUM | Testar com viewport 1080x1920, ajustar padding |
| PNG file size > 1MB | LOW | Usar PNG compression, quantize colors se necessário |
| Font rendering inconsistente | MEDIUM | Fallback para system fonts, testar em múltiplos sistemas |
