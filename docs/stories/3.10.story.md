## 📝 Dev Notes

### Implementation Strategy

**OutputFormatter Service** consolidates outputs from 5 generation services into a unified JSON structure:

1. **Caption Selection** — Prioritizes by type (hook > teaser > cta) then confidence score
2. **Hashtag Normalization** — Maps generator output properties (hashtag, recommendation_score, etc.)
3. **Media Encoding** — Converts image Buffers to base64 strings for JSON serialization
4. **Carousel Aggregation** — Maintains slide metadata (dimensions, slide numbers) during consolidation
5. **Component Tracking** — Records which components are available (for partial outputs)

### Key Design Decisions

- **Base64 Encoding**: Images stored as base64 strings instead of raw buffers for JSON compatibility
- **Partial Support**: Handles missing components gracefully (caption-only, hashtags-only, etc.)
- **Validation Status**: Differentiates between complete/partial/empty outputs
- **Type Safety**: Proper TypeScript imports prevent type conflicts with hashtag fields

### Validation Rules

- **Critical**: Missing caption/media, invalid base64, carousel structure errors
- **Warning**: Exceeded caption length (2200 chars), low confidence (<0.5), flagged hashtags
- **Auto-Selection**: Best caption chosen by type priority + confidence score
