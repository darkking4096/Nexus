### 2. Backend Integration

**New Endpoint**:

**POST /api/profiles/connect**
```json
Request: {
  "username": "user_instagram",
  "password": "...",
  "display_name": "Marina Clientes"
}
Response: {
  "profile_id": "...",
  "instagram_username": "user_instagram",
  "followers_count": 15000,

```

**Error Cases**:
- 400: Invalid credentials
- 401: 2FA required (handle later)
- 403: Not a Business/Creator account
- 429: Rate limited by Instagram

  5. Armazena no database com tokens encriptados

## ⚠️ Considerações Especiais

1. **Segurança de Credenciais Instagram**: 
   - Usar AES-256-GCM para armazenar (não plaintext)
   - Considerar usar app password vs. user password
   - Instagram detecta bots — usar Instagrapi com delays aleatórios

2. **Session Persistence**:
   - Instagrapi mantém session (cookies, user_id, device_id)
   - Armazenar session JSON encriptado para reutilizar
   - Não fazer login toda vez (slow, risca de bloquear)

3. **Rate Limiting**:
   - Instagram limita 200 requests/hora por access token
   - Implementar queue e backoff exponencial

4. **2FA**:
   - Não suportamos 2FA yet (future story)
   - Informar usuário de forma clara
