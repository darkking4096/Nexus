## 📋 Descripción

Implementar funcionalidade de benchmark comparativo que analisa as métricas do perfil em relação aos concorrentes. ## 📚 Dev Notes

### Dependencies Integration
- **SearchService (Story 1.5)**: Fornece método `searchCompetitors(handles: string[])` que retorna dados de concorrentes estruturados
- **AnalyticsService (Story 1.6)**: Fornece método `getProfileMetrics(profileId: string)` para coletar métricas próprias
- **Database**: Usar tabelas `metrics` + `competitors` criadas em Story 1.6

[Source: Story 1.5 (SearchService) and Story 1.6 (AnalyticsService)]

### Algoritmo de Comparação
1. Coletar métricas do profile via `AnalyticsService.getProfileMetrics()`
2. Se `competitorHandles` não passado, buscar via contexto do profile armazenado
3. Coletar métricas de cada competitor (pode usar SearchService para validar handles)
4. Calcular média aritmética das métricas dos competitors
5. Calcular máximo (best) de cada métrica
6. Calcular percentual de diferença: `((profile - avg) / avg) * 100`
7. Retornar estrutura BenchmarkResult

[Source: docs/architecture/FULL-STACK-ARCHITECTURE.md - Analytics Layer]

### Tratamento de Erros
- Se profile não encontrado → Return 404
- Se nenhum competitor com dados → Return 400 com mensagem clara
- Se timeout coleta competitors → Use cache se disponível, fallback graceful
- Ownership validation: Garantir que usuario logado own o profile

[Source: Story 1.6 (AnalyticsService error patterns)]

### Testing Strategy
- Mock SearchService.searchCompetitors()
- Mock AnalyticsService.getProfileMetrics()
- Teste cálculo de percentuais (edge cases: 0%, negativo, grande variância)
- Teste com 1, 2, 3+ competitors
- Teste tratamento de errors e edge cases

[Source: docs/architecture/testing-strategy guidelines]
